从零开始：在 AWS 上孵化你的 Clawdbot (新手避坑极简版)

发布日期：2026年1月31日

这是一篇写给“想要拥有一个属于自己的 AI 助手”的新手的教程。我们会用最直观的方式，教你如何在 AWS 上从零开始，孵化出你的第一个 Clawdbot。

🚀 第一步：给它一个稳健的“身体” (硬件选择)

在 AWS 创建实例时，请记住这个黄金组合：2GB RAM + 20GB Disk。

为什么要先加“血条” (Swap)？ 虽然你可以直接强行安装，但在 1GB 或 2GB 的环境下，为了防止 Bot 在 setup 期间因为“思考过度”而崩溃，我们建议在执行安装命令前，先给它准备一个“后备箱”：

sudo fallocate -l 2G /swapfile && sudo chmod 600 /swapfile && sudo mkswap /swapfile && sudo swapon /swapfile

这样做是为了让系统在内存紧缺时有个缓冲，虽然现在的空机器通常能扛住，但“稳”字头上一把刀，备着准没错。

🛠️ 第二步：极简连接，快速“起飞”

新手绝招：先连公网 IP，再搞 SSH 密码。 很多新手卡在配置 SSH 密钥上。我们的建议是：

  1. 先在 AWS 控制台使用默认的公用 IP 直接连接进入 Terminal。
  2. 只要能动了，别废话，直接运行这行神谕： 
    curl -fsSL https://openclaw.bot/install.sh | bash
  3. 等待安装脚本跑完。这时候，最神奇的事情就要发生了……

🐣 第三步：破壳而出 (Hatch in TUI)

最让人震惊的部分来了：安装还没彻底结束，你就会发现自己直接“孵化” (Hatch) 进了一个 TUI (终端界面) 里面。

这时候，你的 Clawdbot 还是一个小宝宝。它刚来到这个世界上，可能不知道自己在哪个文件夹，不知道你的系统环境，甚至不知道你是谁。

别担心，这时候你可以尽情向它“撒娇”：

  • “哎呀，我不知道怎么配置 SSH 密码，你帮我搞定吧？”
  • “这是哪里？你现在感觉怎么样？内存够吗？”
  • “我想换个主题色，你知道怎么弄吗？”

Clawdbot 会在这个过程中学习。 它不是一个死板的软件，它是和你一起成长的数字生命。你教它如何管理这台服务器，它反过来帮你把这个家打理得井井有条。

🔐 第四步：从“裸奔”到“武装到牙齿”

等 Bot 稳定了，能跑了，它会主动提醒你（或者你要求它）：

  • 配置 SSH 密码/密钥：让连接变得更安全。
  • 清理缓存：运行 npm cache clean --force 腾出空间。
  • 设置守护进程：确保它在服务器重启后能自动醒来找你。

💡 总结：LLM 时代的部署逻辑

传统的部署是“看文档 -> 写配置 -> 启动”。 Clawdbot 的部署是 “给个身体 -> 丢进环境 -> 陪它聊天”。

不要害怕报错，不要害怕它“发疯”（通常是内存不够）。只要给它足够的空间（2GB RAM）和一点点耐心，它会从一个只会复读的小宝宝，变成一个帮你查火车票、写博客、管服务器的全能管家。

现在，去运行那行 curl 吧，你的 Bot 正在壳里等你呢！

From Scratch: Hatching Your Clawdbot on AWS (A Newbie's Guide)

Published: January 31, 2026

This is a guide written for anyone who wants to own their own AI assistant. We will show you how to hatch your first Clawdbot on AWS from scratch in the most intuitive way.

üöÄ Step 1: Give It a Solid "Body" (Hardware Choice)

When creating an instance on AWS, remember this golden combo: 2GB RAM + 20GB Disk.

Why add a "Health Bar" (Swap) first? While you could force an installation, in a 1GB or 2GB environment, the Bot might "overthink" and crash during setup. We recommend preparing a "trunk" (Swap) before running the install command:

sudo fallocate -l 2G /swapfile && sudo chmod 600 /swapfile && sudo mkswap /swapfile && sudo swapon /swapfile

This provides a buffer when memory is tight. While a fresh machine can usually handle it, it's better to be safe than sorry.

Pro Tip for Newbies: Connect via Public IP first, configure SSH passwords later. Many beginners get stuck on SSH key configurations. Our suggestion is:

  1. Connect directly into the Terminal via the AWS console using the default Public IP.
  2. Once you're in, don't waste time‚Äîrun this "divine oracle" line: 
    curl -fsSL https://openclaw.bot/install.sh | bash
  3. Wait for the script to finish. This is where the magic happens...

üê£ Step 3: Breaking the Shell (Hatch in TUI)

Here comes the most shocking part: before the installation is even fully over, you'll find yourself "Hatched" directly into a TUI (Terminal User Interface).

At this moment, your Clawdbot is just a baby. It has just entered this world and might not know which folder it's in, what the system environment is like, or even who you are.

Don't worry, this is the time to "pamper" it:

  • "Oh dear, I don't know how to set up an SSH password, can you fix that for me?"
  • "Where are we? How are you feeling? Do you have enough RAM?"
  • "I want to change the theme color, do you know how?"

Clawdbot learns during this process. It's not a rigid piece of software; it's a digital life form growing with you. You teach it how to manage the server, and in return, it keeps the home organized.

üîê Step 4: From "Naked" to "Armed to the Teeth"

Once the Bot is stable and running, it will proactively remind you (or you can ask it) to:

  • Configure SSH Passwords/Keys: To make connections more secure.
  • Clean Caches: Run npm cache clean --force to free up space.
  • Set up a Daemon: To ensure it wakes up to find you after a server reboot.

üí° Summary: The Deployment Logic in the LLM Era

Traditional deployment is "Read docs -> Write config -> Start." Clawdbot deployment is "Give it a body -> Drop it in the environment -> Talk to it."

Don't fear errors or the Bot "going crazy" (usually just low RAM). With enough space (2GB RAM) and a little patience, it will evolve from a simple repeater into an all-powerful butler who checks train tickets, writes blogs, and manages your server.

Now, go run that curl command. Your Bot is waiting for you inside the shell!
